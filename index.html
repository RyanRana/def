DEF

Dynamic Engagement Framework

A framework that adapts to any web game. Using attention-based reinforcement learning, it modifies live games in real time—speed, rewards, difficulty—to maximize dopamine and engagement, without changing game source code.

Why it matters

Static difficulty misses the mark: too easy and attention drifts; too hard and players leave. DEF runs entirely in the browser. It reads a 1 Hz stream of vitals, input, and score, then uses attention-based RL to nudge only gameplay levers—maximizing engagement and that “one more run” feeling while leaving the game code untouched.

How the agent works

DEF adapts to any web game. Everything runs in the browser; game HTML and scripts stay unchanged. The agent reads signals, infers engagement, and adjusts live game parameters to maximize dopamine and engagement.

Profiles define state creation (what to measure each tick) and mod creation (safe parameters and get/set hooks). At runtime, attention-based reinforcement learning drives the loop: Thompson sampling + attention weighting probe which mods help; a UCB1 bandit then exploits what works to keep players in flow.

Profiles can be generated from game code analysis, so new games can be onboarded without hand-writing state or mod hooks.

Under the hood

The agent runs entirely in the client. Game HTML and JS are never edited—only a small loader is injected so the agent can attach to any web game.

  • Tracker — 1 Hz tick; builds an 18-vector (HR, BR, score, deaths, input, key rhythm, face cues) plus game state.
  • Inference — VitalLens (local) or OpenCV + bandpass FFT for HR/BR and simple face movement/valence.
  • Stimulus — Thompson sampling over mod arms with attention; outputs a short mod plan each tick.
  • Attention-based RL — Classifies engagement (flow / bored / frustrated / excited / disengaged), applies UCB1 over policies and the stimulus plan to modify live game parameters and maximize engagement; restricted to an allowlist (no layout or caps).
  • Mods — Per-game registry of get/set hooks; original game code is left as-is.
